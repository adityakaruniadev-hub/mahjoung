# PRD Collaboration Task Document

## Mission
Create a comprehensive Product Requirements Document (PRD) for a Multiplayer Mahjong Website.

## Collaboration Process

### Agent 1: Coder Agent
**Role:** Technical feasibility, architecture decisions, implementation constraints
**Focus:**
- Tech stack choices
- Scalability concerns
- Real-time multiplayer complexities
- Game state management

**Critical Questions to Address:**
1. What Mahjong variant should we support? (Chinese Classical, Hong Kong, Japanese Riichi, American, etc.)
2. Real-time synchronous or async play option?
3. Web-only or mobile app required?
4. Guest play vs registered users?
5. AI opponents for practice/single-player?
6. Ranking/ELO system? Tournaments?
7. In-game currency? Real money gambling? (LEGAL CONCERNS HERE)

### Agent 2: Business Analytics Agent
**Role:** Market viability, user acquisition, monetization, competitive analysis
**Focus:**
- Target audience assumptions
- Revenue model viability
- User retention strategies

**Critical Questions to Address:**
1. Who is the target audience? (casual players, competitive players, specific regions)
2. What is the competitive landscape?
3. What monetization models work in this space?
4. What are realistic user acquisition costs?

## PRD Structure Required
1. Executive Summary
2. Problem Statement & Market Opportunity (with data, not assumptions)
3. Target User Personas (validated, not invented)
4. Core Features & Functionality (each with acceptance criteria)
5. Technical Architecture (with alternatives considered)
6. Game Mechanics & Rules (standard vs variations)
7. Multiplayer & Real-time Systems (WebSocket? P2P? Server authoritative?)
8. User Flows & UX Considerations
9. Monetization Strategy (with justification)
10. Success Metrics (KPIs)
11. Risk Analysis & Mitigation
12. Open Questions (what we still need to know)

## Deliverable
A comprehensive PRD.md file saved to the workspace.
